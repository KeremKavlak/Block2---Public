Front-end

  • Our game is a maze game, so the ecommerce website will include different products helping the user play the game for the money spent. The screenshots obove show the products, basket and checkout for the user. The customer can sign up and log in to track his/hers purchases and can edit their details after signing up if needed to.

Product search

  • The search box is implemented into javascript into in the shop.php(line 20-30) which call the search function (line 107-127).
  • Using AJAX the javascript updates the html file comunicating to the find_customers.php file posting the criteria entered into the search box

Search result storing

  • Clicking on the low to high button in the shop.php file will redirect into a sort1.php and find the values of prices in the shoping products in ascending order sort1.php(line 17)
  • Clicking on the highto low button in the shop.php file will redirect to sort1.php and find the values of prices in the shoping products in descending order sort2.php(line 17)

Shopping basket

  • Clicking on the Add:, in the shop.php(line 50-52) sends the id and product name to basket.js, inside basket.js a session storage is made for a basket.
  • User can decide to empty the basket and will delete the session storage basket.js(line 60-65).
  • Checkout.php is where the magic happens with making a new collection of purchases in the database.

Customer

Log in/out

  • In the singup.php all the validation and form box is based. Using ajax the data is posted to the server and is posted to singing.php
  • Once in the database user can log in, the code for these functions are in login.php which combine into check-login.php, and customer_login.php
  • The logout.php is called if the user presses the logout button, this destroys the current session.

Edit details

  • After logging in the user details can be changed, this is called by the fech_customer.php code. and posts the information to save_customer.php which overwrites the current information stored in the database.
